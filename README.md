# CS-Core-Engine

**Core CS, Engineered.**

A production-minded knowledge repository exploring how computer systems work under the hood.

This repository goes beyond university notes and exam preparation. It connects core Computer Science subjects to real-world software engineering, system design, backend development, cybersecurity, and machine learning through structured engineering deep dives.

Each topic is documented with:
- Concept explanations
- Architecture diagrams
- Code demonstrations
- Real-world applications
- Interview preparation questions
- Performance analysis and tradeoffs
- Personal learning notes and observations

## Why This Repository Exists

Many students study concepts only to pass exams. The goal of this repository is different:

- Understand how systems actually work
- Connect academic theory to production systems
- Build strong interview-ready fundamentals
- Document engineering knowledge in a structured way
- Create a long-term reference for projects, internships, and system design preparation

This repository serves as both a personal knowledge base and a public record of continuous technical learning.

## Repository Structure

### 01. Low-Level Internals
Understanding how software interacts with operating systems, runtime environments, and memory.

**Core topics:**
- JVM Memory Management
- Python Memory Internals
- Processes vs Threads
- Scheduling and Context Switching
- Garbage Collection
- CPU-bound vs I/O-bound workloads

### 02. Data and Databases
Exploring how databases store, retrieve, optimize, and protect data.

**Core topics:**
- B+ Tree Indexing
- Query Optimization
- Transaction Isolation Levels
- Database Locking
- ACID Properties
- Deadlocks and Recovery

### 03. Networking Systems
Understanding what happens when systems communicate across networks.

**Core topics:**
- The Journey of a Packet
- DNS Resolution
- TCP vs UDP
- TLS Handshake
- HTTP/1.1 vs HTTP/2
- Network Performance Fundamentals

### 04. Algorithms and Design
Connecting data structures and algorithms to real software systems.

**Core topics:**
- Algorithmic Tradeoffs
- Sorting Internals
- Timsort Analysis
- Hash Tables
- Consistent Hashing
- Queues and Message Brokers
- Distributed System Foundations

### 05. Applied Machine Learning
Understanding machine learning from both mathematical and engineering perspectives.

**Core topics:**
- Gradient Descent
- Feature Engineering
- Data Leakage
- Model Evaluation
- Training Pipelines
- Production ML Considerations

## Learning Philosophy

The focus is not memorization.

The focus is understanding:
- Why systems are designed this way
- What tradeoffs engineers make
- How theory becomes production software
- How core concepts appear in real-world projects

This repository is built with an engineering-first mindset: learning by connecting concepts, writing explanations, creating diagrams, and validating ideas through code.
